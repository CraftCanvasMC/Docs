:::note

This guide assumes you are able to use a Gradle wrapper of some capacity, either Gradle Kotlin DSL
or Gradle Groovy DSL. We do not include guides for getting weaver setup on Maven.

:::

**weaver** is a fork of **paperweight** used for custom Canvas build tooling.
**weaver** includes 3 modules, one of them being **weaver-userdev**.
This module provides easy access to Mincraft server internals (NMS) using Gradle.

## Is This Important to You?

Accessing server internals is not necessary for majority of Canvas plugins.
Using the Canvas API or the one of the APIs of our upstream projects (i.e. Paper or Folia) is always preferrable when it can be used.


In some situations, the Canvas API may not be enough and you may need to access server internals directly.
**weaver-userdev** serves as a EULA and license safe way for us to provide you with Minecraft server internals bundled with Canvas' patches and API.

As of **Minecraft 26.1**, Minecraft internals are no longer obfuscated so class names from **weaver-userdev** will be exactly as they appear at runtime.


## Getting Started

If you've decided that you must use Minecraft internals for your project, this guide is here to assist you in adding **weaver-userdev** to your Gradle build script.


### Prerequisites

#### Gradle 9.+

For this guide, we'll be using gradle 9.+. If your Gradle wrapper version starts with the number **9**, you're good to go and you can skip this part!

#### Java 21 & 25

Different Minecraft versions may require different Java SDKs.
Changing your JDK might be different depending on your IDE and setup.
For IntelliJ IDEA users, you can change your JDK by going to **Project** > **Project Structure** and selecting the proper JDK from the **SDK dropdown**.

- **Minecraft 1.21 - 1.21.11**: Java 21
- **Minecraft 26.1+**: Java 25


### weaver Setup

There are two ways to go about adding **weaver-userdev** to your project. Both methods will work just fine!
